# [1주차 Reflection] - 아이디어톤 1일차

## 아이디어톤 진행방법

- 디자인씽킹 프로세스 이해 → 코딩으로 구현가능한 불편사항 도출 → 불편사항 구체화를 위한 아디어 현실화 → 프로토타입 제작 → 시연 및 발표 → 역량진단 평가



## Design Thinking 활용

### 디자인씽킹을 활용한 문제발견

| 단계                    | 과정                                 |
| ----------------------- | ------------------------------------ |
| 공감 (Empathize)        | 관찰, 인터뷰, 체험으로 문제점을 발견 |
| 문제 정의 (Define)      | 문제점에 대해 정의하고 분석하는 단계 |
| 아이디어 도출 (Ideate)  | 아이디어를 자유롭게 발산하는 과정    |
| 시제품 제작 (Prototype) | 아이디어를 구현하는 과정             |
| 테스트 (Test)           | 피드백을 통해 아이디어를 개선        |

- 공감의 세가지 방법
  - 관찰 : 사로운 관점에서 관찰, 여럿이 함께 관찰도 좋음
  - 인터뷰 : 고객에게 느낌, 좋은 점, 불편한 점 등을 질문하여 정리
  - 직접체험 : 문제속으로 직접 들어가 경험, 진짜 문제를 발견할 수 있음
- 관찰의 Tool : 공감지도, 고객여정맵, 페르소나 만들기
- 공감 : 고객이 진심으로 원하는 것이 무엇인지 파악하고 문제를 발견하는 시작점이 '공감'이다
- 관점 서술문 작성
